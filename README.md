# 🤖 RAG-ассистент с ChromaDB и кешированием

Рабочий пример ИИ-ассистента с технологией RAG (поиск по базе знаний), с локальным векторным хранилищем ChromaDB и кешированием ответов.


## Бизнес ценность
Ассистент ускоряет работу HR-отдела и технической поддержки в 5 раз, за счёт кеширования запросов среднее время ответа уменьшается в половину.

## 📋 Описание

Этот проект демонстрирует, как работает RAG-система:

1. **Пользователь задает вопрос**
2. **Проверка кеша** - если ответ уже есть, возвращается из кеша
3. **Поиск релевантных документов** - векторный поиск в ChromaDB
4. **Формирование контекста** - сборка найденных фрагментов
5. **Генерация ответа** - запрос к LLM (OpenAI) с контекстом
6. **Кеширование ответа** - сохранение для будущих запросов
7. **Возврат ответа пользователю**

## 🎯 Особенности

- ✅ **Локальное векторное хранилище** с ChromaDB
- ✅ **OpenAI эмбеддинги** для высокого качества поиска
- ✅ **Поддержка ProxyAPI** — альтернатива прямому OpenAI API для пользователей из РФ
- ✅ **Семантический поиск** по документам
- ✅ **Кеширование ответов** для экономии API-запросов
- ✅ **Telegram бот** с поддержкой эскалации вопросов администратору
- ✅ **Простая архитектура** с понятным кодом
- ✅ **Подробные комментарии** для обучения
- ✅ **Три режима работы**: интерактивный, демонстрационный и Telegram бот


- ✅ Оплата в рублях (карты РФ, СБП, крипта)
- ✅ Не требует VPN
- ✅ Доступ к моделям: GPT-4o, GPT-4o-mini, GPT-4-turbo и др.
- ✅ Те же цены, что и у OpenAI (без наценки)

Получить ключ можно на [proxyapi.ru](https://proxyapi.ru/)

### 5. Настройка Telegram бота (опционально)

Для запуска Telegram бота добавьте в `.env`:

```env
# Токен Telegram бота (получите у @BotFather)
TELEGRAM_BOT_TOKEN=your_telegram_bot_token_here

# ID администратора для эскалации сложных вопросов (получите у @userinfobot)
TELEGRAM_ADMIN_ID=your_telegram_admin_id_here
```

**Как работает эскалация:**
1. Пользователь задаёт вопрос боту
2. Если бот не находит ответ в базе знаний → предлагает отправить question администратору
3. При согласии пользователя → бот отправляет вопрос на указанный `TELEGRAM_ADMIN_ID`
4. Администратор получает уведомление с деталями (вопрос, пользователь, время)

**Запуск в режиме Telegram бота:**
```bash
# Установите переменную окружения
export BOT_MODE=telegram  # Linux/Mac
set BOT_MODE=telegram     # Windows

python main.py
```

Или просто выберите режим 3 в интерактивном меню.

## 🔧 Архитектура

### Модуль `cache.py`

Реализует простое кеширование на основе хеша запроса:

```python
cache = ResponseCache()

# Проверка кеша
answer = cache.get(query)

# Сохранение в кеш
cache.set(query, answer)
```

**Зачем кеш?**
- Экономия денег на API-запросах
- Мгновенные ответы на повторные вопросы
- Снижение нагрузки на LLM

### Модуль `embeddings.py`

Работа с векторным хранилищем:

```python
store = EmbeddingStore()

# Добавление документов
store.add_documents(documents)

# Поиск релевантных фрагментов
results = store.search(query, top_k=3)
```

**Используемые технологии:**
- ChromaDB - векторная база данных
- OpenAI Embeddings (text-embedding-3-small) - создание эмбеддингов
- Chunking - разбивка текстов на фрагменты с перекрытием

### Модуль `rag.py`

Основная логика RAG:

```python
assistant = RAGAssistant(embedding_store)

# Генерация ответа с RAG
answer, sources = assistant.generate_response(query)
```

**Процесс RAG:**
1. Векторный поиск релевантных документов
2. Формирование контекста из найденных фрагментов
3. Создание промпта для LLM
4. Генерация ответа с учетом контекста

### Модуль `main.py`

Оркестрация всех компонентов:

```python
# Инициализация
cache = ResponseCache()
store = EmbeddingStore()
assistant = RAGAssistant(store)

# Ответ на вопрос
cached = cache.get(query)
if not cached:
    answer = assistant.generate_response(query)
    cache.set(query, answer)
```

### Модуль `prompt_loader.py`

Загрузка системных промптов из файлов в папке `prompts/`:

```python
from prompt_loader import PromptLoader

# Инициализация загрузчика
prompt_loader = PromptLoader(prompts_dir="prompts")

# Получение всех промптов
all_prompts = prompt_loader.get_all_prompts()

# Получение конкретного промпта по ключу
system_prompt = prompt_loader.get_prompt("hr_assistant")

# Объединение всех промптов
combined_prompt = prompt_loader.get_combined_prompt()
```

**Зачем нужны промпты?**
- Настройка стиля и тональности ответов ассистента
- Определение правил поведения и ограничений
- Добавление специфичных для предметной области инструкций
- Легкое изменение поведения без изменения кода

### Модуль `document_loader.py`

Автоматическая загрузка документов из папки `knowledge_base/`:

```python
from document_loader import DocumentLoader

# Инициализация загрузчика
doc_loader = DocumentLoader(knowledge_base_dir="knowledge_base")

# Загрузка всех документов
documents = doc_loader.load_all_documents()

# Загрузка документов определенного типа
pdf_docs = doc_loader.get_documents_by_type("pdf")
txt_docs = doc_loader.get_documents_by_type("txt")
```

**Поддерживаемые форматы:**
- TXT - текстовые файлы
- CSV - табличные данные (каждая строка как отдельный документ)
- JSON - структурированные данные
- PDF - текст из PDF файлов (требует PyPDF2)
- DOCX - документы Word (требует python-docx)
- MD/MARKDOWN - файлы разметки

**Автоматическая обработка:**
- Извлечение текста из различных форматов
- Сохранение метаданных об источниках
- Рекурсивный обход подпапок
- Пропуск системных и временных файлов

## 📊 Технические детали

### Эмбеддинги

- **Модель**: text-embedding-3-small (OpenAI)
- **Размерность**: 1536
- **Языки**: поддержка множества языков, включая русский
- **Преимущества**: 
  - Высокое качество семантического поиска
  - Отличная работа с многоязычными текстами
  - Не требует загрузки локальной модели
- **Недостатки**:
  - Требует API запросы (платно)
  - Нужно интернет-соединение

### Chunking

- **Размер чанка**: 500 символов
- **Перекрытие**: 50 символов
- **Причина**: сохранение контекста на границах

### ChromaDB

- **Тип**: персистентное хранилище
- **Метрика**: cosine similarity
- **Расположение**: ./chroma_db/

### Кеш

- **Тип**: JSON файл
- **Ключ**: SHA-256 хеш нормализованного запроса
- **Расположение**: ./cache.json

## ⚠️ Известные ограничения

1. **API ключ OpenAI** требуется для работы. Используется как для генерации ответов, так и для создания эмбеддингов. Запросы платные.

2. **Стоимость API**: 
   - Создание эмбеддингов: ~$0.02 за 1M токенов (text-embedding-3-small)
   - Генерация ответов: ~$0.50-$15 за 1M токенов (в зависимости от модели)
   - При первой загрузке документов будет сделано несколько запросов для создания эмбеддингов

3. **Кеш** простой и не учитывает изменения в базе знаний. При обновлении документов очистите кеш командой `clear_cache`

4. **Интернет-соединение** необходимо для всех операций (эмбеддинги и генерация ответов)
